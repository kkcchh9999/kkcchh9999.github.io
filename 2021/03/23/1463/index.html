    1로 만들기

    작성일 2021-03-23 Edited on 2021-03-28 In 알고리즘

    문제

    1로 만들기, 1463번

    정수 X에 사용할 수 있는 연산은 다음과 같이 세 가지 이다.

    1. X가 3으로 나누어 떨어지면, 3으로 나눈다.
    2. X가 2로 나누어 떨어지면, 2로 나눈다.
    3. 1을 뺀다.

    정수 N이 주어졌을 때, 위와 같은 연산 세 개를 적절히 사용해서 1을 만들려고 한다. 연산을 사용하는 횟수의 최솟값을 출력하시오.

    입력 : 첫째 줄에 1보다 크거나 같고, 106보다 작거나 같은 정수 N이 주어진다.

    풀이 과정

    DP[N] 이 연산을 사용하는 최솟값이라고 하자. 이 때, DP[N]은 DP[N-1] + 1로 만들 수 있다.
    만약 N이 3으로 나누어 떨어지거나, 2로 나누어 떨어지면 DP[N] = DP[N/3]+1 혹은 DP[N/2]+1 이 될 수도 있다. 하지만 DP[N]은 최소의 수를 나타내야한다. 따라서 DP[N] = min(DP[N-1]+1, DP[N/2]+1, DP[N/3]+1) 을 만족해야 한다. 이를 코드로 나타낸다.

    코드

    #include <iostream>

using namespace std;

int DP[1000001];

int main() {

    int n = 0;

    cin >> n; 

    for (int i = 2; i <= n; i++)
    {
        DP[i] = DP[i - 1] + 1;
        if (i % 3 == 0)
            DP[i] = min(DP[i], DP[i / 3] + 1);
        if (i % 2 == 0)
            DP[i] = min(DP[i], DP[i / 2] + 1);
    }
    cout << DP[n] << endl;
}

    소감

    알고리즘 카테고리의 첫 문제라고 봐도 될 만큼 이전의 입출력과는 난이도가 확연히 달라진것 같다. 알고리즘 공부만 해도 엄청난 노력이 필요할 것 같다. 아무래도 C++공부는 알고리즘과 별개로 이후에 더 진행해야 할 것 같다.

    혼자 1시간 반정도를 고민하다가 도저히 알 수 없어서 답을 찾아보고 풀이했다. DP 자체를 처음 접하는지라 어떻게 풀어야하는지 전혀 감을 잡지 못하고 있었다고 봐도 될 것 같다. 나의 접근 방식은 완전히 잘못 된 것이었다. 이번 문제를 풀고 난 후에, DP의 정의와 DP 문제의 풀이법을 찾아보고 정리해 보았다.

    Dynamic Programin(동적 계획법)의 첫 문제이다. DP는 큰 문제를 작은 문제로 나누어 해결하는 것으로 분할정복법과 비슷하지만, 반복의 유무에따라 작은 부분이 반복된다면 DP, 그렇지 않으면 분할정복이라고 볼 수 있다. DP는 작은 문제들의 반복이라, 다음번 반복해야 할 때에는 앞서 계산했던 값을 사용하여 계산한다. 따라서 앞에서 계산한 부분을 저장해 놓을 필요가 있다.
