1, 2, 3 더하기

작성일 2021-03-30 Edited on 2021-04-06 In 알고리즘

문제

Hello World, 2557번

정수 4를 1, 2, 3의 합으로 나타내는 방법은 총 7가지가 있다. 합을 나타낼 때는 수를 1개 이상 사용해야 한다.

  • 1+1+1+1
  • 1+1+2
  • 1+2+1
  • 2+1+1
  • 2+2
  • 1+3
  • 3+1

정수 n이 주어졌을 때, n을 1, 2, 3의 합으로 나타내는 방법의 수를 구하는 프로그램을 작성하시오.

입력 : 첫째 줄에 테스트 케이스의 개수 T가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 n이 주어진다. n은 양수이며 11보다 작다.

출력 : 각 테스트 케이스마다, n을 1, 2, 3의 합으로 나타내는 방법의 수를 출력한다.

풀이 과정

DP[n]이 n에 대한 방법의 수 일때 DP[n]은 DP[n-1] + 1, DP[n-2] + 2, DP[n-3] + 3 이 세 가지의 방법으로 만들어 질 수 있다. 따라서 DP[n] = DP[n-1]+1 + DP[n-2]+2 + DP[n-3]+3 이라고 볼 수 있다. 이를 이용하여 풀이한다.

코드

#include <iostream>

using namespace std;

int main() {

    int DP[11] = { 0, };
    int t = 0,n = 0;
    DP[1] = 1;
    DP[2] = 2;
    DP[3] = 4;

    cin >> t; 
    for(int j = 0; j<t; j++){
        cin >> n; 
          for (int i = 4; i <= n; i++)
        {
            DP[i] = DP[i - 1] + DP[i - 2] + DP[i - 3];
        }
        cout << DP[n] << endl;
    }
}

소감

결국 이 문제도 DP라는 큰 틀 안에 있어서 풀이 방법이 이전에 풀었던 문제들과 비슷한 것 같다. 하지만 DP라고 알고 있었다고 해도 풀이 방법을 찾는데 생각보다 오래 걸렸다. 좀 더 많은 유형을 풀어보고 문제의 n을 구하겠다는것 보다 DP 문제라는 큰 장르를 분석해서 해결할 수 있도록 해야겠다.

2 x n 타일링

작성일 2021-03-29 Edited on 2021-03-30 In 알고리즘

문제

2 x n 타일링 2, 11727번

2×n 직사각형을 1×2, 2×1과 2×2 타일로 채우는 방법의 수를 구하는 프로그램을 작성하시오.
아래 그림은 2×17 직사각형을 채운 한가지 예이다.

입력 : 첫째 줄에 n이 주어진다. 출력 : 둘째 줄에 2×n 크기의 직사각형을 채우는 방법의 수를 10,007로 나눈 나머지를 출력한다.

풀이 과정

이전에 풀었던 타일링과 같은 방법이지만, DP[n-2]에서 1x2 두장을 더하는 방법과, 2x2 한장을 더하는 방법까지 두 가지의 방법이 생겼기 때문에, DP[n] = DP[n-1] + (DP[n-2] x 2)를 이용하여 풀 수 있다.

코드

#include <iostream>

using namespace std;

int main() {

    int DP[1001] = { 0, };
    int n = 0, answer = 0;

    cin >> n; 
    DP[1] = 1;
    DP[2] = 3;
    for (int i = 3; i <= n; i++)
    {
        DP[i] = (DP[i - 1] + (DP[i - 2] * 2))%10007;
    }
    cout << DP[n] << endl;
}

소감

처음으로 DP 문제를 해답을 보지 않은 상태로 해결했다. 하지만 이는 이전 문제에서 약간만 바뀌었을 뿐임으로 자만하지 않고 계속 실력을 정진해 나갈 필요가 있어 보인다.

2 x n 타일링

문제

2 x n 타일링, 11726번

2×n 크기의 직사각형을 1×2, 2×1 타일로 채우는 방법의 수를 구하는 프로그램을 작성하시오.
아래 그림은 2×5 크기의 직사각형을 채운 한 가지 방법의 예이다.

입력 : 첫째 줄에 n이 주어진다. 출력 : 둘째 줄에 2×n 크기의 직사각형을 채우는 방법의 수를 10,007로 나눈 나머지를 출력한다.

풀이 과정

DP[n]은 방법의 수라고 할 때, DP[1] 은 2x1 한장으로 이루어지고, DP[2] 는 1x2 두장 혹은 2x1 두장으로 이루어진다. DP[3] 은 DP[1]에 1x2 타일 두장을 쌓은 모양을 더한 방법과, DP[2]에 2x1 타일 한장을 더한 방법으로 이루어진다.
즉, DP[n] 은 DP[n-2]에 1x2 타일 두장을 쌓은 모양을 더한 방법과, DP[n-1]에 2x1 타일 한장을 더한 방법으로 이루어진다. 이를 10007로 나누어 가면서 DP 배열에 저장, DP[n]을 찾는다.

코드

#include <iostream>

using namespace std;

int main() {

    int DP[1001] = { 0, };
    int n = 0;

    cin >> n; 
    DP[1] = 1;
    DP[2] = 2;
    for (int i = 3; i <= n; i++)
    {
        DP[i] = (DP[i - 1] + DP[i - 2])%10007;
    }
    cout << DP[n] << endl;
}

소감

이번 문제도 혼자 해결하지 못했다. n-1, n-2와의 관계로 해결하면 될 것 같다는 해결법에는 가까이 갔지만 블럭의 모양이다보니 겹치는 부분이 있을거라 생각하여 오히려 해답에서 멀어졌다. 조금 더 문제 풀이 수를 늘려서 이정도 문제는 쉽게 해결할 수 있도록 해야겠다.

1로 만들기

작성일 2021-03-23 Edited on 2021-03-28 In 알고리즘

문제

1로 만들기, 1463번

정수 X에 사용할 수 있는 연산은 다음과 같이 세 가지 이다.

  1. X가 3으로 나누어 떨어지면, 3으로 나눈다.
  2. X가 2로 나누어 떨어지면, 2로 나눈다.
  3. 1을 뺀다.

정수 N이 주어졌을 때, 위와 같은 연산 세 개를 적절히 사용해서 1을 만들려고 한다. 연산을 사용하는 횟수의 최솟값을 출력하시오.

입력 : 첫째 줄에 1보다 크거나 같고, 106보다 작거나 같은 정수 N이 주어진다.

풀이 과정

DP[N] 이 연산을 사용하는 최솟값이라고 하자. 이 때, DP[N]은 DP[N-1] + 1로 만들 수 있다.
만약 N이 3으로 나누어 떨어지거나, 2로 나누어 떨어지면 DP[N] = DP[N/3]+1 혹은 DP[N/2]+1 이 될 수도 있다. 하지만 DP[N]은 최소의 수를 나타내야한다. 따라서 DP[N] = min(DP[N-1]+1, DP[N/2]+1, DP[N/3]+1) 을 만족해야 한다. 이를 코드로 나타낸다.

코드

#include <iostream>

using namespace std;

int DP[1000001];

int main() {

    int n = 0;

    cin >> n; 

    for (int i = 2; i <= n; i++)
    {
        DP[i] = DP[i - 1] + 1;
        if (i % 3 == 0)
            DP[i] = min(DP[i], DP[i / 3] + 1);
        if (i % 2 == 0)
            DP[i] = min(DP[i], DP[i / 2] + 1);
    }
    cout << DP[n] << endl;
}

소감

알고리즘 카테고리의 첫 문제라고 봐도 될 만큼 이전의 입출력과는 난이도가 확연히 달라진것 같다. 알고리즘 공부만 해도 엄청난 노력이 필요할 것 같다. 아무래도 C++공부는 알고리즘과 별개로 이후에 더 진행해야 할 것 같다.

혼자 1시간 반정도를 고민하다가 도저히 알 수 없어서 답을 찾아보고 풀이했다. DP 자체를 처음 접하는지라 어떻게 풀어야하는지 전혀 감을 잡지 못하고 있었다고 봐도 될 것 같다. 나의 접근 방식은 완전히 잘못 된 것이었다. 이번 문제를 풀고 난 후에, DP의 정의와 DP 문제의 풀이법을 찾아보고 정리해 보았다.

Dynamic Programin(동적 계획법)의 첫 문제이다. DP는 큰 문제를 작은 문제로 나누어 해결하는 것으로 분할정복법과 비슷하지만, 반복의 유무에따라 작은 부분이 반복된다면 DP, 그렇지 않으면 분할정복이라고 볼 수 있다. DP는 작은 문제들의 반복이라, 다음번 반복해야 할 때에는 앞서 계산했던 값을 사용하여 계산한다. 따라서 앞에서 계산한 부분을 저장해 놓을 필요가 있다.

별 찍기-12

문제

별 찍기-12, 2522번

예제를 보고 규칙을 유추한 뒤에 별을 찍어 보세요.
입력 : 3

풀이 과정

이 문제 또한 다른 별찍기와 크게 다르지 않지만, 이 문제를 풀 때는 위와 아래로 분리하지 않고, 하나의 for 안에서 윗부분과 아랫부분을 모두 출력했다.
입력이 n이고 for 루프의 i가 1 ~ 2n-1 일때, 각 줄에는 |n-i|개의 공백이 출력되고, n- |n-i|개의 별이 출력된다. 이를 이용해 해결했다.

코드

#include <iostream>

using namespace std;

int main() {

    int n;
    cin >> n;

    for (int i = 1; i <= n * 2 - 1; i++)
    {
        for (int j = 0; j < abs(n - i); j++)
            cout << " ";
        for (int j = 0; j < n - abs(n - i); j++)
            cout << "*";

        cout << endl;
    }
}

소감

이 문제의 해답은 처음에 위 아래를 분리하여 풀고 난 후에, 어떻게하면 한번에 위 아래를 다 출력할 수 있을지 답을 찾아봐서 알게 된 경우이다. 이 경우에 코드가 더 깔끔해지는 것 같고, 절대값을 표기하는 abs라는 함수도 알게되었다. 경우에 따라 절대값도 활용할 수 있도록 시야를 넓혀 갈 필요가 있겠다.

이 문제 이후에도 별 찍기 문제를 몇문제 더 풀었지만, 난이도도 그렇게 높지 않고, 새로운 C++ 함수를 알게 된 문제도 아니라서 포스팅은 하지 않고, 여기까지 입출력 문제에 해당하는 문제는 모두 풀었다. 다음 포스팅부터는 DP에 해당하는 문제이다.

최소, 최대

작성일 2021-03-22 Edited on 2021-03-28 In 알고리즘

문제

최소, 최대, 10818번

N개의 정수가 주어진다. 이때, 최솟값과 최댓값을 구하는 프로그램을 작성하시오.

입력 : 첫째 줄에 정수의 개수 N (1 ≤ N ≤ 1,000,000)이 주어진다. 둘째 줄에는 N개의 정수를 공백으로 구분해서 주어진다. 모든 정수는 -1,000,000보다 크거나 같고, 1,000,000보다 작거나 같은 정수이다.

풀이 과정

최소값인 -1000000을 max, 최대값인 1000000을 min으로 두고, n번의 루프동안 cin으로 입력받는 수를 max와 min 각각 한번씩 비교하여, max보다 크면 max로 저장, min보다 작으면 min으로 저장하여 출력한다.

코드

#include <iostream>

using namespace std;

int main() {

    int n;
    int max = -1000000, min = 1000000;
    int tmp;

    cin >> n;
    
    for (int i = 0; i < n; i++)
    {
        cin >> tmp;
        if (max < tmp)
        {
            max = tmp;
        }
        if (min > tmp)
        {
            min = tmp;
        }
    }
    cout << min << " " << max << endl;

}

소감

최대 1000000개의 입력이 들어오기 때문에, 모든 입력을 다 받은 후에 비교하는것은 불가능해보였고, 입력이 들어올때마다 비교하여 저장하고 출력하는 방식을 선택했다. 이제 슬슬 1학년때 처음 C언어를 배우던 때의 과제 정도의 수준을 갖는 문제가 나오고 있는 것 같다. 입력값의 범위를 잘못 보아 한번 틀렸는데, 문제를 잘 읽도록 해야겠다.

문제

합, 8393번

n이 주어졌을 때, 1부터 n까지 합을 구하는 프로그램을 작성하시오.

풀이 과정

단순히 for루프를 이용하여 1n까지 더하면 O(n)이 된다. 하지만 약간의 수학을 이용하여 1n 까지의 합은 n(n+1) / 2 인 점을 이용하면 O(1)로 만들수 있다.

코드

#include <iostream>

using namespace std;

int main() {

    int n;
    cin >> n;
    cout << (n * (n + 1) / 2);
}

소감

이제 C++언어가 조금씩은 익숙해 진 것 같다. 이제 조금씩 더 좋은 알고리즘을 생각하기 위해 노력할 만한 문제들이 나오지 않을까 기대가 된다.

2007년

문제

2007년, 1924번

오늘은 2007년 1월 1일 월요일이다. 그렇다면 2007년 x월 y일은 무슨 요일일까? 이를 알아내는 프로그램을 작성하시오.

입력 : 첫째 줄에 빈 칸을 사이에 두고 x(1 ≤ x ≤ 12)와 y(1 ≤ y ≤ 31)이 주어진다. 참고로 2007년에는 1, 3, 5, 7, 8, 10, 12월은 31일까지, 4, 6, 9, 11월은 30일까지, 2월은 28일까지 있다.

출력 : 첫째 줄에 x월 y일이 무슨 요일인지에 따라 SUN, MON, TUE, WED, THU, FRI, SAT중 하나를 출력한다.

풀이 과정

일주일은 7일이고, 이를 이용해 나머지 연산을 통하여 문제를 해결한다. x월일 경우, (1월의 일 수(31) + 2월의 일 수(28)+….(x-1)월의 일 수(??) + y )을 하면 총 몇일인지 변환이 가능하고, 여기에 % 7 연산을 한 결과에 따라 요일을 출력하는 방식으로 해결한다.

코드

#include

using namespace std;

int main() {

int x, y, tmp = 0;
cin >> x >> y;

for (int i = 1; i < x; i++)
{
    if (i == 1 || i == 3 || i == 5 || i == 7 || i == 8 || i == 10 || i == 12)
        tmp = tmp + 31;
    else if (i == 2)
        tmp = tmp + 28;
    else
        tmp = tmp + 30;
}
tmp = tmp + y;
if (tmp % 7 == 1)
    cout << "MON" << endl;
else if (tmp % 7 == 2)
    cout << "TUE" << endl;
else if (tmp % 7 == 3)
    cout << "WED" << endl;
else if (tmp % 7 == 4)
    cout << "THR" << endl;
else if (tmp % 7 == 5)
    cout << "FRI" << endl;
else if (tmp % 7 == 6)
    cout << "SAT" << endl;
else
    cout << "SON" << endl;

}

```
이와 같은 방법을 사용했었다. 하지만 오답이 나왔고, 왜 틀린건지 알 수 없어서 답을 찾아보았다. 결론부터 말하자면 처음 사용한 문제 해결 방법이 틀리지는 않았다. 다만 요일을 표기하는 알파벳이 오타가 있어서 오답 처리되었다. 덕분에 답을 찾게 되면서 배열을 사용하는 더 깔끔한 방법을 찾을 수 있었다. 비교 연산의 수가 줄게 되니 속도도 더 빨라질 것 같다. 앞으로는 정답을 맞추더라도 답을 찾아보아야겠다.
